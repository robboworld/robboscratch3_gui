# Сценарии (готовые проекты)

Всё настраивается **в этой папке**: положите `.sb3`, картинку превью и отредактируйте **`manifest.json`**.

## `manifest.json`

Файл — JSON-массив (или объект с полем `"scenarios": [ ... ]`). Каждый элемент:

| Поле | Обязательно | Описание |
|------|-------------|----------|
| `sb3` или `file` | да | Имя файла проекта **в этой папке**, например `urok-1.sb3`. Можно путь от корня статики: `static/scenarios/urok-1.sb3`. |
| `preview` или `image` | желательно | Превью карточки: файл в этой папке (`cover.png`) или путь `static/...`. Рекомендуется ~4:3 (PNG, JPG, SVG, WebP). |
| `title` или `name` | да | Заголовок на карточке и имя проекта в редакторе после открытия. |
| `titleRu` | нет | Заголовок по-русски (показывается при русской локали GUI). |
| `description` | нет | Короткий текст на карточке (латиница / универсальный). |
| `descriptionRu` | нет | Тот же текст по-русски (показывается при русской локали GUI). |
| `id` | нет | Уникальный id для аналитики; иначе генерируется из заголовка. |
| `tags` | нет | Массив строк для фильтров, по умолчанию `["robbo"]`. |

Пример минимальной записи:

```json
{
    "sb3": "moy-proekt.sb3",
    "preview": "moy-proekt.png",
    "title": "Урок 1. Движение",
    "titleRu": "Урок 1. Движение",
    "description": "Robot moves forward using encoders.",
    "descriptionRu": "Робот едет вперёд по энкодерам.",
    "tags": ["robbo", "lesson"]
}
```

После изменения `manifest.json` обновите страницу редактора (или снова откройте **Файл → Scenarios**).

## Файлы

- Держите `.sb3` и превью рядом с `manifest.json` или указывайте путь через префикс `static/...`.
- В репозитории может лежать `sample-preview.svg` / `sample-scenario.sb3` как образец; замените их своими.
